## What We're Building

A read-only "This Year's Plan" summary card on the Dashboard that shows the user a concrete, actionable breakdown of what to do in the current year. Plus a print/save button to export it.

## Task List

### 1. Add "This Year" card to Dashboard

**Location:** Insert into `renderDashboard()` function, positioned AFTER the Plan Health verdict (`verdictHtml`) and Big Levers (`leversHtml`) sections, but BEFORE the stats row (`stats-row`).

**Data source:** Use the first year of the projection array (`proj[0]`) — all needed data is already computed by `runProjection()`. No engine changes needed.

**Card structure:**
- Collapsible card with a header "This Year's Plan (Age XX)" where XX = `p.currentAge`
- Default state: expanded
- Toggle button (chevron) to collapse/expand
- Store collapse state in `state.thisYearCollapsed` (default false)

**Content to display — pull these from `proj[0]`:**

Income section:
- Earned income (if still working): `proj[0].earned` — label "Employment Income"
- Social Security: `proj[0].ss` — label "Social Security"
- SSDI income: `proj[0].ssdi` — label "SSDI Benefits" (only show if > 0)
- Pension income: `proj[0].pensionIncome` — label "Pension Income" (only show if > 0)

Withdrawals section:
- Total withdrawal: `proj[0].withdrawal` — label "Portfolio Withdrawals"
- Show which account it comes from based on withdrawal order and amounts. The projection stores `proj[0].preTax`, `proj[0].roth`, `proj[0].cash`, `proj[0].taxableInv` — compare year 0 and year 1 balances or use the withdrawal breakdown if available.
- If RMD applies: `proj[0].rmd` — label "Required Minimum Distribution (RMD)" with note "(included in pre-tax withdrawal)"

Roth Conversions section (only if enabled):
- Conversion amount: `proj[0].rothConv` — label "Roth Conversion"
- Conversion tax cost: `proj[0].conversionTax` — label "Tax on Conversion"

Taxes & Medicare section:
- Federal tax: `proj[0].tax` — label "Estimated Federal Tax"  
- State tax: `proj[0].stateTax` — label "Estimated State Tax" (only show if > 0)
- Medicare premium: show base Part B cost — label "Medicare Premium" (only show if age >= 65)
- IRMAA surcharge: `proj[0].irmaa` — label "Medicare Surcharge (IRMAA)" (only show if > 0)

Expenses section:
- Total expenses: `proj[0].expenses` — label "Total Living Expenses"
- Charitable giving: `proj[0].charitableAmt` or equivalent — label "Charitable Giving" (only show if > 0)
- QCD amount if applicable — label "Qualified Charitable Distribution (QCD)" (only show if > 0)
- Debt payments: `proj[0].debtPayments` — label "Debt/Liability Payments" (only show if > 0)

Bottom line:
- Net cash flow or total income vs total outflow summary
- Show "Your money lasts through age XX" as reassurance

**Important:** Only show line items that are non-zero or relevant to the user's situation. Don't show SSDI, pension, IRMAA, QCD, etc. if they're $0. Keep it clean.

### 4. Print/Save button

**Location:** Small button in the card header, next to the collapse toggle.

**Behavior:** Generates a clean, printable one-page summary. Two implementation options (pick the simpler one):

**Option A (preferred): Print-friendly popup**
- Opens a new window with a clean, minimal HTML page
- White background, no dark theme, clear typography
- Formatted for printing (proper margins, no overflow)
- User can Cmd+P / Ctrl+P to print or save as PDF
- Include: RetIQ branding (subtle), date generated, all the same data from the card
- Auto-trigger print dialog with `window.print()`

**Option B (fallback): Copy to clipboard**
- Formats the summary as clean plain text
- Copies to clipboard with a "Copied!" confirmation
- User can paste into email, notes, etc.

**Implement Option A.** The print output should look professional — something you could hand to an accountant or financial advisor. Include a small disclaimer at the bottom: "Generated by RetIQ (retirementiq.app). This is a projection based on your inputs, not financial advice."
